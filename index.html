I'm a software engineer at . I work on local and cloud build systems.


I like to build simple tools that make humans do things better. Prior to Apple I pursued implementing a cloud spreadsheet system with rich values inside cells (images, sounds, videos).


I have not been working on rich spreadsheets since I joined Apple, however this problem domain is very dear to my heart. Engineering-wise, rich spreadsheets have overlaps with my work as a cloud build system engineer.


I'm deeply enthused about programming environments, plus design and implementation of programming languages. Back in grad school, I rediscovered the idea of “rich spreadsheets”, which dates back to 1986 [1], but is somehow not materialized in major spreadsheet environments today.


Implementing a rich calc engine [2] made me appreciate how core underlying primitives of a build system intersect with ones of a calc engine [3], and I find myself lucky that I'm in the vicinity of people who hack on Xcode, llbuild and llbuild2. :)

  1. Object-oriented spreadsheets: the analytic spreadsheet package (by Kurt W. Piersol published in OOPSLA'86)
  2. https://github.com/johari/minicell
  3. Build Systems à la Carte (ICFP'18)
