Back in grad school, I rediscovered the idea of “rich spreadsheets” (dating back to 1986 [1]). Modern spreadsheet products (Excel and Google Sheets) are slowly adopting some of these ideas.. but I think there is room for a lot more.


I prototyped a new spreadsheet system focusing on rich values inside cells, utilizing build system techniques to extend the range of tasks that can be tackled at an interactive rate using spreadsheets.


I have not been working on rich spreadsheets since I joined Apple, and my work as an intern in Excel team was focused on "Gridlets". However implementing a polished "rich spreadsheet system" is a problem very dear to my heart.


Engineering-wise, rich spreadsheets have overlaps with my work as a cloud build system engineer. Implementing a rich calc engine [2] made me appreciate how core underlying primitives of a build system intersect with ones of a calc engine [3].

  2. https://github.com/johari/minicell
